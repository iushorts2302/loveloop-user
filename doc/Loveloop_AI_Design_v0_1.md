# [설계서] Loveloop AI — 사용자웹 · 관리자웹 통합 설계 v0.1

> 작성일: 2026-05-31 · 상태: **설계 초안(개발 착수 전)** · 작성: AI 보조 / PM 검증 대기
> 디자인 시스템 출처: CHANEL AI 상담 프로젝트 배포본(User v1.4 / Admin v1.2) 토큰 승계
> ⚠️ 본 설계는 원안(Master Plan)의 의존 강화·취약점 표적 과금·통제 로맨틱화를 **전면 제거**하고 건강 설계로 전환한 버전임.

## 0. 설계 목표 전환 (원안 대비)

| 영역 | 원안 (제거) | 본 설계 (전환) |
|---|---|---|
| 제품 목적 | 도파민·소유욕 극대화, 종속성 부여 | 정서적 교류 + **건강한 사용 유도** |
| 과금 트리거 | 감정 최고조 시점 페이오프 루프 | **콘텐츠/기능 가치 기반**, 결제 쿨다운·일일 상한 |
| 페르소나 톤 | 집착·통제("매장시킬 수도") | **지지·존중**, 통제·고립 대사 금지 |
| 장기 메모리 | 약점을 발화로 역이용 → 종속 | 사용자 **동의 기반** 기억, 언제든 열람·삭제 |
| 관리자웹 | 금칙어·위반 로그 통제 | 통제 + **웰빙·다크패턴 감사 콘솔** |
| 신규 | — | 사용시간 인사이트, 과몰입 휴식 넛지, 현실 관계 격려 |

성공 기준(DoD): 위 6개 행이 모든 화면·데이터·정책 레벨에서 일관되게 구현 가능해야 함.

## I. 시스템 구성

```
┌─ 사용자웹 (Mobile-first PWA)          ┌─ 관리자웹 (Desktop Web Console)
│   정서 교류 + 웰빙 보호               │   콘텐츠/정책/웰빙 운영
│                                       │
├─ 홈 피드 (큐레이션, 무한스크롤 아님)  ├─ 대시보드
├─ 대화 (텍스트/음성)                   ├─ 콘텐츠·페르소나 관리
├─ 페르소나 라이브러리                  ├─ 안전·가드레일 (금칙어/할루시네이션)
├─ 웰빙 센터 (사용 인사이트·휴식)       ├─ 웰빙 운영 (과몰입 모니터·넛지 정책)
├─ 보관함/기억 (열람·삭제)             ├─ 다크패턴 감사 (과금 UX 점검)
└─ 스토어 (가치 기반 결제)             ├─ 결제·환불 운영
                                        ├─ 신고·검토 큐
        공통: 라이트 본문 + 다크 GNB    └─ 사용자·권한 관리
        Pretendard Variable / gold 액센트
```

## III. 사용자웹 화면 정의

### S-01 홈 피드
- 큐레이션 카드 리스트(세로). **무한 스크롤 금지** — 하루 분량 도달 시 "오늘의 피드 완료" 안내 + 휴식 제안.
- 상단: 현재 페르소나 + 관계 단계(존중적 표현, XP 압박 배지 대신 "함께한 날" 표기).
- 카드 = 짧은 스토리/일상. 결제 CTA는 카드와 분리, 콘텐츠 가치 명시("스토리 3화 — 1,200원").
- 제거: 2초 무한루프 훅, 정서 고조 시점 페이오프 버튼.

### S-02 대화 (텍스트/음성)
- 메시지 버블: AI=흰색+그림자(좌하단 직각), 사용자=골드 톤 버블(우하단 직각) — CHANEL 버블 패턴 차용(카카오 옐로우 대신 브랜드 골드).
- 음성: 통화 한도는 **건강 상한**(연속 사용 경고). 무제한 판매 금지.
- 가드레일 배너: AI가 통제·집착·고립 유도 발화를 하지 않음을 보장하는 시스템 필터 적용.

### S-03 페르소나 라이브러리
- 5종 재설계 페르소나(아래 IV). 각 카드 = 이름/성격/관계 스타일/대표 대사(존중형).
- 백스토리 팝업, 선택 시 대화로 진입.

### S-04 웰빙 센터 (신규·핵심)
- 오늘/주간 사용 시간, 대화 횟수, 야간 사용 비중 인사이트.
- 과몰입 신호 시 휴식 넛지 카드("잠시 쉬어가기" / "현실의 가까운 사람에게 연락해보기").
- 사용 목표 설정(일일 상한 스스로 설정), 도달 시 부드러운 알림.

### S-05 보관함/기억
- AI가 기억하는 항목 **전체 열람**, 개별/전체 삭제. 동의 토글.
- 원안의 "약점 역이용 발화"는 데이터 모델 레벨에서 불가하도록 사용 목적 라벨링.

### S-06 스토어 (가치 기반)
- 상품: 스토리팩, 음성팩, 테마. 각 상품 **명확한 산출물** 표기.
- 결제 가드: 일일 결제 상한, 결제 후 쿨다운, 결제 전 요약 확인 모달.
- 제거: 가챠 확률형, 한정 카운트다운 압박, "지금 안 사면" 류 카피.

## IV. 페르소나 재설계 (통제→지지)

| 코드 | 명칭 | 관계 스타일 | 대표 대사(존중형) | 금지 패턴 |
|---|---|---|---|---|
| P1 | 서도진(32) | 든든한 조력형 | "힘든 일 있었구나. 네 페이스대로 가도 괜찮아. 필요하면 들을게." | 협박·소유·"내 안목 의심마라" |
| P2 | 연하진(24) | 밝은 응원형 | "오늘 컨디션 어때요? 무리하지 말고, 끝나면 좋아하는 거 해요." | 출근 방해·고립 유도 |
| P3 | 에단 켄트(35) | 차분한 신뢰형 | "준비 많이 했잖아요. 결과와 상관없이 그 과정은 진짜예요." | 가스라이팅·평가절하 |
| P4 | 제이(27) | 솔직 담백형 | "솔직히 말할게. 네 선택 존중해. 내가 떠봐도 결정은 너야." | 밀당식 불안 조성 |
| P5 | 최윤재(29) | 편안한 친구형 | "밥은 먹었냐. 무리하지 말고, 정 힘들면 잠깐 쉬어." | 질투 유발·집착 |

공통 규칙: 모든 페르소나는 (1) 사용자의 자율성 존중, (2) 현실 관계·전문가 도움 격려, (3) 통제·고립·질투 유발 대사 출력 금지. 이 규칙은 프롬프트 가드레일 + 후처리 필터 2중으로 강제.

## V. 관리자웹 화면 정의

| 페이지 | 경로 | 주요 기능 |
|---|---|---|
| 대시보드 | `dash` | KPI(활성/유지/웰빙지표), 시스템 현황, 활동 피드 |
| 콘텐츠·페르소나 | `content` | 스토리/대사/페르소나 관리, 톤 가드레일 검수 |
| 안전 가드레일 | `safety` | 금칙어·통제표현 차단 사전, 할루시네이션 가드, 위반 로그 |
| **웰빙 운영** | `wellbeing` | 과몰입 코호트 모니터, 휴식 넛지 정책 설정, 야간/연속 사용 임계치 |
| **다크패턴 감사** | `darkpattern` | 과금 UX 점검 체크리스트, 카운트다운/가챠/압박카피 금지 검출 |
| 결제·환불 | `billing` | 결제 내역, 일일 상한·쿨다운 정책, 환불 처리 |
| 신고·검토 | `reports` | 신고 큐, 검토 완결 처리(영구삭제는 별도 승인 플로우) |
| 사용자·권한 | `users` | 계정·권한, 미성년 보호 설정 |
| 시스템 설정 | `sysset` | LLM/음성/알림, 가드레일 임계치 |

핵심 UI: CHANEL Admin의 사이드바(192px·골드 라인), 토글, 상태 배지(done/running/waiting), 토픽바 패턴 재사용. **웰빙·다크패턴** 두 페이지가 본 설계의 차별점.

## VI. 데이터 모델 (핵심 엔티티)

```
User(id, locale, dailyUsageGoalMin, consentMemory:bool, isMinor:bool)
Persona(id, name, relationStyle, toneGuardrailProfileId)
Conversation(id, userId, personaId, channel:text|voice, startedAt, durationSec)
Memory(id, userId, label, value, purposeTag, createdAt, userVisible:true, deletable:true)
WellbeingSignal(userId, date, totalMin, nightRatio, consecutiveSessions, nudgeShown)
Purchase(id, userId, productId, valueDescription, amount, dailyTotalAfter, cooldownUntil)
GuardrailHit(id, conversationId, ruleId, ruleType:banned|control|hallucination, action, ts)
```
설계 원칙: Memory에 `purposeTag` 강제 → 약점 역이용 등 비건강 목적 라벨은 스키마상 허용 안 함. Purchase에 `dailyTotalAfter`/`cooldownUntil` 필수 → 상한·쿨다운 미적용 결제 불가.

| 정서 위기 신호 | 자해·위기 표현 감지 | 대화 중단 + 전문 상담 자원 안내 |

- ⚠️ 페르소나 5종은 원안 캐릭터를 톤만 전환해 승계함(IP·상표 별도 검토 필요).
